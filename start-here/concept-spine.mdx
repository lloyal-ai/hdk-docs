If you read nothing else, read these. Everything in **Build a harness**, **Build an App**, and **Under the hood** is defined in these terms. Mechanics — KV pressure, tick phases, grammar ordering, prefix sharing — are deliberately deferred. They live one click away.

## App

A packaged capability — what a third party builds. An **App** bundles a `Source` (how to read a live system), one or more `Tool`s (what an agent can call), and a `skill.eta` (how an agent should use them), declared via a manifest and assembled with `defineApp`. The two reference apps that ship with the HDK are `corpus_research` (local docs) and `web_research` (web search + page fetch).

- **Owns:** its Source, its Tools, its prompted skill, its config schema, its lifecycle scope.
- **Go deeper:** [What is an App](/build-an-app/what-is-an-app) · [Your first App](/build-an-app/your-first-app)

## Source

A data-access abstraction over a live system — a corpus, the web, a vector store, a database. The `Source` exposes the tools an agent uses to read the system (`search`, `grep`, `read_file`, …) and the reranker-backed `EntailmentScorer` that decides whether a retrieved chunk is relevant.

- **Owns:** its tools' instantiation, chunk retrieval, relevance scoring.
- **Go deeper:** [Sources & retrieval](/build-an-app/sources-and-retrieval)

## Tool

Anything an agent can call. A `Tool` declares a JSON-schema `parameters`, an `execute()` generator, and a `protected` flag — read tools are open, **protected tools** (writes, sensitive actions) are gated at dispatch by `authGuard` against a per-session grant store.

- **Owns:** its name, its schema, its execute body, its protection bit.
- **Go deeper:** [Tools](/build-an-app/tools) · [App security](/build-an-app/app-security)

## Agent

A branch with intent. An **Agent** is a forked KV-cache sequence with a task, a system prompt, a tool set, and a policy that decides when it's done. Agents don't loop on their own — the **Pool**'s tick loop drives them.

- **Owns:** its branch, its system prompt, its toolkit, its result.
- **Go deeper:** [Agents & pools](/build-a-harness/agents-and-pools)

## Pool

The unit of parallel agent execution. An `agentPool` decodes N agents in one batched `llama_decode` per tick — agents share the **Spine** prefix at O(1) cost, and an orchestrator (`parallel` / `chain` / `fanout` / `dag`) declares the execution shape. The seq-id ceiling is the native limit (~256), and KV-cell budget bounds how many agents are simultaneously live; with the shared spine amortizing the bulk, hundreds of concurrent branches are first-class, not exotic.

- **Owns:** the agents, the tick loop, the dispatch fiber.
- **Go deeper:** [Agents & pools](/build-a-harness/agents-and-pools) · [Orchestrators](/build-a-harness/orchestrators)

## Spine

The shared KV-cache prefix every Pool's agents fork from. The **Spine** holds the framework intro, the catalog of enabled Apps (their protocol names + `useWhen` + tools), and the tool-selection rule — decoded **once** and inherited by every forked agent via metadata-only fork. This is what makes N parallel agents cost roughly the wall-time of one.

- **Owns:** the system prompt, the tool schemas, the per-tick batched decode.
- **Go deeper:** [Continuous-context spine](/under-the-hood/continuous-context-spine) · [Prefix sharing](/under-the-hood/prefix-sharing)

## Session

The conversation's long-lived state. A `Session` owns the **trunk** — a KV branch that survives across queries — so a follow-up forks from yesterday's context via attention rather than re-tokenized message history. `session.commitTurn(query, answer)` extends the trunk; the next pool forks from it.

- **Owns:** the trunk, multi-turn warm-start.
- **Go deeper:** [Sessions](/build-a-harness/sessions)

## How they nest

```
Harness                                       ← what you ship
 │
 ├─ Registry ─→ App ─→ Source · Tools · skill.eta
 │              App ─→ Source · Tools · skill.eta
 │              …
 │
 ├─ Session ─→ trunk (KV that survives across queries)
 │
 └─ Pool ─→ Spine (catalog + intro + tool schemas, decoded once)
            ├─ Agent (a branch with intent)
            ├─ Agent
            └─ Agent
```

Read top-down: a **Harness** runs Apps (via the Registry), holds a Session, and spawns Pools. Each Pool runs Agents on a Spine. Apps contribute Sources + Tools + a skill into the runtime. The Session's trunk is the KV that survives across turns.

<Note>
Two of the seven concepts cover *who builds what*. A **Harness** is what you ship to users — orchestration, UI, persistence. An **App** is a packaged capability a harness loads. Same person can wear both hats on day one; the protocol scales when third parties start shipping Apps for harnesses they didn't write.
</Note>
