---
title: Quick start
description: Scaffold a multi-agent research harness, point it at a local model, run it. One model file, no API keys, no accounts.
---

Three commands from an empty directory to a planned, parallel, source-grounded answer — running on your hardware. The only thing you need that isn't `npx`-able is a model file.

## 1 · Init

```bash
npx harness.dev my-harness
cd my-harness
npm install
```

That scaffolded a complete, working harness — not a stub:

- `src/main.ts` — boot: load the model, build the AgentApp registry, run the query.
- `src/harness.ts` — the orchestrator: plan the query into tasks, fan them out as parallel agents, synthesize the reports into one answer.
- `prompts/plan.eta`, `prompts/synth.eta` — the planner and synthesizer prompts, yours to edit.
- `harness.json` — config: model path and context size.

It ships with the `lloyal/wikipedia` reference AgentApp preinstalled from the signed channel — a keyless research capability, so there is nothing to sign up for. Full file-by-file breakdown: [Scaffolding](/cli/scaffolding).

## 2 · Point it at a model

Download a GGUF — [Qwen3.5-4B](https://huggingface.co/unsloth/Qwen3.5-4B-GGUF) at `Q4_K_M` (~2.5 GB) is the reference choice — and put its path in `harness.json`:

```json
{
  "model": {
    "path": "/path/to/Qwen3.5-4B-Q4_K_M.gguf",
    "nCtx": 32768
  }
}
```

(Or skip the config and pass `--model /path/to/model.gguf` on each run.) A machine with **8 GB of RAM and any GPU** (Metal / CUDA / Vulkan) runs this comfortably; the model download is the slow part.

## 3 · Run

```bash
npm run dev "What was the Cuban missile crisis?"
```

Watch what happens:

1. A **planner** decomposes the question into a handful of orthogonal research tasks.
2. **Agents fan out in parallel** — one per task, each a fork of the same model context, decoding together in one GPU batch on your machine.
3. Each agent researches through the Wikipedia AgentApp's tools and reports its findings.
4. A **synthesizer** reads every report and composes the answer that prints to your terminal.

All inference happened locally — the only network traffic was the Wikipedia REST calls the tools made. No inference API, no keys, no per-token bill.

<Note>
The scaffold assumes a **thinking model** (Qwen3.5-family), which is what the runtime defaults expect. If you swap in a non-thinking `-Instruct` model and tool calls start appearing as plain prose instead of dispatching, that's the `enableThinking` flag — see [Agents and pools](/build-a-harness/agents-and-pools#im-running-a-thinking-mode-model--enablethinking).
</Note>

## 4 · Play

The harness is yours. Four moves, in rising order of ambition:

**Reshape the voice.** Open `prompts/synth.eta` and change what the synthesizer produces — bullet briefings, ELI5, a skeptic's summary with confidence levels. Re-run. No rebuild, no redeploy.

**Reshape the workflow.** Open `src/harness.ts` — the entire orchestration is ~100 lines, and the harness IS the orchestrator: there's no hidden agent pulling strings. Swap the `parallel` fan-out for a `chain` (each step reads the previous step's findings through shared attention) or a `dag` (arbitrary dependency graphs). [Orchestrators](/build-a-harness/orchestrators) has the shapes.

<Tip>
The harness code is written in Effection generators — `function*` and `yield*` where you'd expect `async`/`await`. If that's new to you, [Thinking in Lloyal](/start-here/thinking-in-lloyal) is a 10-minute read that makes every code block in these docs legible. Read it before editing `harness.ts`.
</Tip>

**Search your own files.** Add the corpus AgentApp and the planner starts routing tasks to your documents:

```bash
npx harness.dev install lloyal/corpus
```

```ts
// in src/main.ts, next to the wikipedia app:
import { createCorpusApp } from "@lloyal-labs/corpus-app";

yield* configStore.set("corpus", { corpusPath: "/path/to/your/docs" });
yield* registry.enable(createCorpusApp);
```

**Build your own capability.** `npx harness.dev app my-app` scaffolds an AgentApp with working Wikipedia-backed tools — swap them for your database, your API, your filesystem. [Your first AgentApp](/build-an-app/your-first-app) walks the contract.
