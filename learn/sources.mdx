A source is any system your agents can search. File system. Web. Vector database. Email inbox. JIRA. Internal wiki. Anything with content and a way to find it.

You define the tools. The harness orchestrates agents with those tools via [agentPool()](/learn/agents#parallel-agents-createagentpool).

## The contract

A source is a data backend. It provides tools and chunks — not orchestration.

```typescript
import { Source } from "@lloyal-labs/lloyal-agents";
import type { Tool } from "@lloyal-labs/lloyal-agents";
import type { Operation } from "effection";

class MySource extends Source<MyContext, MyChunk> {
  readonly name = "jira";                            // 1. Label

  get tools(): Tool[] {                              // 2. Data-access tools
    return [this._searchTool, this._readTool];
  }

  *bind(ctx: MyContext): Operation<void> {           // 3. Late-bind runtime deps
    this._searchTool = new JiraSearchTool(this._client, ctx.reranker);
  }

  getChunks(): MyChunk[] {                           // 4. Post-use chunks for reranking
    return this._buffer.getAll();
  }
}
```

<Steps>
  <Step title="name">
    A label. Used in output to attribute which source produced what. Keep it short — `"web"`, `"corpus"`, `"jira"`.
  </Step>

  <Step title="tools">
    The data access tools this source provides. The harness passes these to `agentPool()` along with its own prompt and orchestration config. The source doesn't decide how agents use the tools — the harness does.
  </Step>

  <Step title="bind(ctx)">
    Called before tools are used. Wire up runtime dependencies — reranker, API clients, database connections — that weren't available at construction time. This is a generator, so you can do async setup.
  </Step>

  <Step title="getChunks()">
    Returns buffered content for post-use reranking. During agent execution, tools fetch content (pages, documents, code files). The source buffers it. After agents complete, the harness calls `getChunks()` and reranks against the original query to select the best passages for synthesis.
  </Step>
</Steps>

<Note>
The source abstraction is intentionally thin. Three members: `name`, `tools`, `bind`. No orchestration, no prompts, no policy. You subclass `Source`, implement the contract, and pass instances to your harness.
</Note>
