The fastest way to learn the AgentApp protocol is to read `createCorpusApp` end-to-end. It's small, real, and every concept the [Build an AgentApp](/build-an-app/what-is-an-app) page introduced shows up exactly once.

This page walks through `packages/apps/corpus/` in the order the loader sees it.

## File layout

```
packages/apps/corpus/
‚îú‚îÄ‚îÄ app.json              ‚Üê manifest: name, protocol, configSchema
‚îú‚îÄ‚îÄ skill.eta             ‚Üê per-spawn agent preamble (Eta template)
‚îú‚îÄ‚îÄ src/
‚îÇ   ‚îú‚îÄ‚îÄ index.ts          ‚Üê AppFactory: createCorpusApp()
‚îÇ   ‚îú‚îÄ‚îÄ source.ts         ‚Üê CorpusSource class
‚îÇ   ‚îî‚îÄ‚îÄ tools/
‚îÇ       ‚îú‚îÄ‚îÄ grep.ts
‚îÇ       ‚îú‚îÄ‚îÄ read_file.ts
‚îÇ       ‚îî‚îÄ‚îÄ search.ts     ‚Üê reranker-scored chunk retrieval
‚îî‚îÄ‚îÄ package.json
```

Three things to keep in mind while reading:

1. **The factory is an Effection generator.** It runs in a detached scope the registry provides. Everything is `yield*` and `Operation<T>`.
2. **Config and reranker arrive via Context, not arguments.** The harness sets `AppConfigStoreCtx` and `RerankerCtx` before enabling the AgentApp; the factory pulls them with `.expect()`.
3. **`defineApp` is called last,** after the source + tools are constructed. Validation runs synchronously and throws on a bad manifest.

## 1. The manifest ‚Äî `app.json`

```jsonc
{
  "name": "corpus",
  "version": "1.0.0",
  "appProtocolVersion": "3.0",
  "protocol": {
    "name": "corpus_research",
    "useWhen": "investigating a local document corpus ‚Äî finding occurrences of terms, reading specific files at line offsets, semantic retrieval over indexed corpus content.",
    "tools": ["grep", "read_file", "search"]
  },
  "configSchema": {
    "type": "object",
    "required": ["corpusPath"],
    "properties": { "corpusPath": { "type": "string" } }
  }
}
```

What each field does:

- **`name`** ‚Äî the package-level identifier the registry keys on. Matches `[a-z][a-z0-9_-]{1,63}`.
- **`protocol.name`** ‚Äî the model-facing identifier rendered into the spine's catalog. Different from `name` so the AgentApp's npm package name and the model-facing routing name can evolve independently. Same regex.
- **`protocol.useWhen`** ‚Äî one sentence the planner reads to decide whether to route a task to this AgentApp. Bounded length, no chat-role markers, no newlines, no code fences. Sanitized at `defineApp` time so AgentApp-supplied strings can't break out of the catalog block.
- **`protocol.tools`** ‚Äî the canonical list of tool names. The tool map passed to `defineApp` must cover exactly this set.
- **`appProtocolVersion`** ‚Äî `"3.0"` (current). The registry refuses AgentApps whose declared version isn't supported.
- **`configSchema`** ‚Äî JSON Schema for the AgentApp's stored config. The registry validates existing config against this on enable. `"x-secret": true` marks credentials (omitted here; the web AgentApp's `tavilyKey` uses it).

## 2. The skill ‚Äî `skill.eta`

The per-spawn agent preamble. The framework renders Eta with an `AgentRenderCtx` plus anything the source supplies via `promptData()`.

```eta
You are a thorough research assistant analyzing a knowledge base to complete your task.
<% if (it.agentCount > 1) { -%>

You are one of <%= it.agentCount %> parallel agents. The other agents are covering:
<%= it.siblingTasks.map(function(q) { return '- ' + q }).join('\n') %>

Stay focused on your own task. The other agents handle the rest.
<% } -%>

Available files:
<%= it.toc %>

You have <%= it.maxTurns %> tool calls. Plan your investigation within this budget.

RULES FOR TOOL OPTIMAL USE:
- If grep returns zero matches, the exact pattern is absent ‚Äî but the topic may still be present. Try broader keywords or search instead.
- If a query returns `Resource unavailable`, another agent already issued that exact call. Try different terms.
- Don't repeat the same query or read the same lines twice.
- When a tool returns an error about time limit, KV limit or word limit, stop and call report() tool with all your findings so far.

PROCESS:
1. Grep or search to find relevant content for your task.
2. Read every matching line with read_file to verify in context.
3. Based on what you read, identify specific claims to verify or details that are missing, then grep/search for those.
4. Call report() with comprehensive findings: direct quotes, data points, source URLs.
```

Notes worth flagging:

- **`it.toc`** comes from `CorpusSource.promptData()`. The source decides what the agent sees about its content surface.
- **Sibling-task awareness.** The `it.agentCount > 1` block teaches the agent it's one of N ‚Äî preventing duplicate searches when a pool fans out.
- **No `Apply the **`.** The framework prepends `BOUNDARY_MARKER(protocol.name)` to this template per spawn. `defineApp` rejects a `skill` that contains the literal `Apply the **` substring (it would emit twice).
- **The `report()` tool** is a *harness-defined* terminal tool, not an AgentApp-defined tool. It's available because the harness threads it into the pool ‚Äî AgentApps describe how their tools compose toward a report, but don't ship the report tool themselves.

## 3. The factory ‚Äî `src/index.ts`

This is the whole file (~50 lines):

```ts
import { readFileSync } from "node:fs";
import { join } from "node:path";
import { call } from "effection";
import type { Operation } from "effection";
import { AppConfigStoreCtx, RerankerCtx } from "@lloyal-labs/lloyal-agents";
import type { App, AppManifest, Tool } from "@lloyal-labs/lloyal-agents";
import { defineApp } from "@lloyal-labs/rig";
import type { Reranker } from "@lloyal-labs/rig";
import { loadResources, chunkResources } from "@lloyal-labs/rig/node";
import { CorpusSource } from "./source";

export function* createCorpusApp(): Operation<App> {
  const dir = join(__dirname, "..");
  const manifest = JSON.parse(readFileSync(join(dir, "app.json"), "utf8")) as AppManifest;
  const skill = readFileSync(join(dir, "skill.eta"), "utf8");

  // 1. Pull the reranker from context. Throw if absent ‚Äî search needs it.
  let reranker: Reranker;
  try {
    reranker = yield* RerankerCtx.expect();
  } catch {
    throw new Error(
      "createCorpusApp: the corpus app requires a reranker. " +
      "Set RerankerCtx via createReranker(...) before enabling.",
    );
  }

  // 2. Read config from the store.
  const cfgStore = yield* AppConfigStoreCtx.expect();
  const cfg = (yield* cfgStore.get("corpus")) ?? {};
  const corpusPath = typeof cfg.corpusPath === "string" ? cfg.corpusPath : undefined;
  if (!corpusPath) {
    throw new Error("createCorpusApp: missing config `corpusPath`.");
  }

  // 3. Heavy setup runs at construction (one shot).
  const resources = loadResources(corpusPath);
  const chunks = chunkResources(resources);
  yield* call(() => reranker.tokenizeChunks(chunks));

  // 4. Construct source + tools.
  const source = new CorpusSource(resources, chunks, reranker);
  const tools: Record<string, Tool> = {};
  for (const t of source.tools) tools[t.name] = t;

  // 5. defineApp validates + assembles.
  return defineApp({ manifest, source, tools, skill });
}
```

Five steps, each load-bearing:

1. **Reranker via `RerankerCtx`.** A required dep ‚Äî the search tool scores chunks. The factory throws if absent so the failure surfaces at registration, not at first search.
2. **Config via `AppConfigStoreCtx`.** The harness owns the store; the factory only reads. Missing `corpusPath` is also a hard error at construction.
3. **Heavy setup once.** Loading the filesystem and tokenizing chunks for the reranker is the expensive part. It happens here, not on every search.
4. **Source then tools.** The `CorpusSource` instance owns the chunks + reranker; `source.tools` gives back ready-to-use `Tool` instances. The factory just collects them into a `Record` keyed by `tool.name`.
5. **`defineApp` last.** Validation runs synchronously. If `Object.keys(tools)` doesn't match `protocol.tools` exactly, or `useWhen` contains a forbidden pattern, or `skill` contains `Apply the **`, this throws before the AgentApp reaches the registry.

## 4. The Source ‚Äî `src/source.ts`

The Source owns chunked data + the reranker, exposes the tool instances, and supplies `promptData()` to the skill template:

```ts
export class CorpusSource extends Source<unknown, Chunk> {
  readonly name = "corpus";

  constructor(
    private resources: Resource[],
    private chunks: Chunk[],
    private reranker: Reranker,
  ) {
    super();
    this._reranker = reranker;
  }

  get tools(): readonly Tool[] {
    return [
      new GrepTool(this.resources),
      new ReadFileTool(this.resources),
      new SearchTool(this.chunks, this.reranker),
    ];
  }

  promptData(): CorpusPromptData {
    return { toc: this._buildToc() };
  }

  private _buildToc(): string {
    // file ‚Üí top-level-headings list, joined per-line ‚Äî see source.ts
  }
}
```

`promptData()` is what the skill template reads as `it.toc`. The corpus surfaces a TOC; the web AgentApp returns nothing here (its skill renders without an `it.toc` block).

[Sources & retrieval](/build-an-app/sources-and-retrieval) covers the `Source` base, `EntailmentScorer`, and how the reranker scores during dispatch.

## 5. The tools ‚Äî `src/tools/`

Each tool is a `Tool<TArgs>` subclass. Three appear in this AgentApp; here's the `SearchTool` shape:

```ts
export class SearchTool extends Tool<{ query: string }> {
  readonly name = "search";
  readonly protected = false;
  readonly description = "Search the knowledge base. Returns sections ranked by relevance with line ranges for read_file.";
  readonly parameters: JsonSchema = {
    type: "object",
    properties: { query: { type: "string", description: "Search query" } },
    required: ["query"],
  };

  *execute(args: { query: string }, ctx?: ToolContext): Operation<unknown> {
    // scores chunks via this._reranker, applies threshold, returns envelope
    return { hits, thresholdScore, totalScored, topRejected };
  }
}
```

- **`name`** matches the manifest's `protocol.tools` entry.
- **`description`** is what the model reads to choose between tools.
- **`parameters`** is JSON Schema ‚Äî the framework turns it into a GBNF grammar so the model can only emit valid arguments.
- **`protected = false`** marks this as a read tool, open to any agent. Write/sensitive tools set `true`; `authGuard` gates them at dispatch. See [AgentApp security](/build-an-app/app-security).
- **`execute`** is the Effection generator the pool calls. The return value is JSON-serialized and prefilled back into the agent's KV at the call position.

The full tool surface (Tool base, `protected`, `probe`, schema auto-generation) lives in [Tools](/build-an-app/tools).

## 6. Using the AgentApp in a harness

Once the factory exists, a harness enables it via `createAppRegistry`:

```ts
yield* RerankerCtx.set(reranker);   // BEFORE ‚Äî the corpus factory reads it
const configStore = createInMemoryConfigStore();
yield* configStore.set("corpus", { corpusPath: "/path/to/docs" });

const registry = yield* createAppRegistry({
  configStore,
  apps: [createCorpusApp, createWebApp],   // build-time factories
});

// registry.enabled() now returns the two enabled Apps.
// registry scope-exit tears them down (factory ensure()s fire).
```

That's the entire boot wiring. The harness's pool reads the registry from `AppRegistryCtx` and composes every enabled AgentApp's tools + skill into its spine ‚Äî see [Agents & pools](/build-a-harness/agents-and-pools).
