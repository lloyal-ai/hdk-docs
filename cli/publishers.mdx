To publish apps to `apps.lloyal.ai` you need a **publisher account** — a handle that identifies your apps in the catalog under `<publisher-handle>/<app-name>`. Registration is one-time per identity; subsequent publishes use the cached account record.

## Register a publisher handle

```bash
npx harness.dev publishers register --handle <handle> [--yes]
```

`<handle>` matches the grammar `[a-z][a-z0-9_-]{1,63}` (see [Scaffolding > Identifier grammar](/cli/scaffolding#identifier-grammar)).

**Example:**

```bash
npx harness.dev publishers register --handle acme

# Publisher ToS (v1): https://docs.lloyal.ai/licensing/publisher-tos
#
# By registering you attest that you have read and accept the publisher Terms of Service.
#
# Accept the publisher ToS? [y/N] y
# registered as acme
#   tosVersion: v1
#   registeredAt: 2026-06-08T02:25:37.976Z
```

Pass `--yes` to skip the interactive ToS prompt (still prints the link — required when running non-interactively):

```bash
npx harness.dev publishers register --handle acme --yes
```

### Reserved handles

Currently only `lloyal` is reserved (for Lloyal Labs first-party apps). Every other handle is first-come-first-served, subject to good-faith use.

If you try to register a reserved handle from a non-authorized identity, the server returns `handle-reserved`:

```bash
$ npx harness.dev publishers register --handle lloyal --yes
# harness.dev publishers register: HTTP 409 Conflict
#   error: handle-reserved
```

### Error codes

| Error | Cause | Fix |
|---|---|---|
| `handle-invalid` | Your handle doesn't match `[a-z][a-z0-9_-]{1,63}` | Use lowercase ASCII; 2-64 chars; no special chars except `_` and `-` |
| `handle-taken` | Another publisher already claimed it | Pick a different handle |
| `handle-reserved` | The handle is reserved (e.g., `lloyal`) | Pick a different handle |
| `tos-required` | You omitted `--yes` or answered `n` at the prompt | Re-run, accept the ToS |
| `already-registered` | Your identity already has a handle (the response includes which one) | Use the existing handle — see `harness.dev publishers me` |
| `service-tokens-cannot-register` | You ran from CI with `CF_ACCESS_CLIENT_*` env vars set | Register interactively first, then CI can publish |

### Idempotency

Re-running with the same handle on an already-registered identity is a no-op:

```bash
$ npx harness.dev publishers register --handle acme --yes
# (same handle as on file)
# registered as acme
#   tosVersion: v1
#   registeredAt: 2026-06-08T02:25:37.976Z  (original date — no changes)
```

Re-running with a *different* handle returns `409 already-registered`. If you genuinely need to transfer ownership to a new handle, contact `support@lloyal.ai`.

## Inspect your account

```bash
npx harness.dev publishers me
```

Returns the publisher record for your current authenticated identity:

```
acme
  status:         active
  published:      3
  tosVersion:     v1
  tosAcceptedAt:  2026-06-08T02:25:37.976Z
  createdAt:      2026-06-08T02:25:37.976Z
```

| Field | Meaning |
|---|---|
| (top line) | Your registered handle |
| `status` | `active` or `suspended` (suspended publishers can't submit; contact support to appeal) |
| `published` | Count of approved versions across all your apps |
| `tosVersion` | Which ToS version you attested to. If you see `tos-version-stale` on subsequent publish, re-run `publishers register` to attest the latest version |
| `tosAcceptedAt` / `createdAt` | Timestamps of your initial registration |

If no record exists for your identity, the command exits non-zero with a hint:

```
harness.dev publishers me: no publisher account for this identity.
Run `harness.dev publishers register --handle <handle>` first.
```

## What the ToS covers

The [Publisher ToS](/licensing/publisher-tos) is short — read it once. The high-level points:

- You retain ownership of your code; you grant Lloyal Labs a license to inspect, sign, and distribute approved tarballs via the channel.
- You agree to submit only non-malicious code that complies with the App protocol and respects third-party licenses.
- Lloyal Labs commits to reviewing every submission, communicating outcomes, and preserving rejected artifacts for 30 days (appeals window).
- The ToS is versioned (`v1`, `v2`, ...). Material revisions require re-attestation (re-run `publishers register`).
